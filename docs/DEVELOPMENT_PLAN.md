# 开发计划

## 总体验收目标

用户通过 OpenClaw 完成首次设置和日常记录；报告生成前收到一次精简的查漏补缺提醒；到点后自动得到一张包含状态、原因、数据完整度和明日行动的日报 PNG，同时本地健康驾驶舱更新。

## M0：干净骨架（已完成）

当前交付：

- 新发布目录、依赖最小的 Python 包和 CLI。
- 外置运行数据目录和版本化 SQLite schema。
- 单 owner 数据边界。
- 独立可安装的 OpenClaw Skill。
- 资产盘点、架构和开发路线。

验收：`init`、`status`、单元测试和 Skill 校验通过，仓库中无运行数据。

## M1：每日记录垂直切片（已完成）

开发内容：

- 实现统一 `checkin` 输入协议和事件类型。
- 迁移时间解释、字段校验、饮食、运动、身体、睡眠、情绪、症状、周期记录。
- 支持修正和删除记录。
- 所有写入保留来源文本、置信度和审计记录。

验收：OpenClaw 能通过 CLI 或本地 HTTP 写入并读取当天记录；无 AI 配置时仍可完成常见打卡。

当前实现采用 CLI 契约；本地 HTTP 仅在后续确有需要时增加，避免为 OpenClaw 重复建设常驻服务。

## M2：首次引导与查漏补缺（已完成）

开发内容：

- 收集目标、基础档案、每日必填偏好、时区和报告时间。
- 实现 `gap-check --date`，区分缺失、未计划和不适用。
- 输出一条最多询问三个关键缺口的补录消息。
- 提供 OpenClaw 定时任务安装说明和可执行命令。

验收：报告前能生成稳定、非指责式的补录提醒；跳过补录不会阻断报告。

当前交付的 `schedule plan` 会输出查漏补缺、日报、周报和月报四类可安装任务；全部报告任务调用已完成的本地报告引擎。

## M3：健康分、置信度与洞察（已完成）

开发内容：

- 将健康状态与数据完整度分开计算。
- 实现运动、饮食、身体、恢复/周期四个模块。
- 实现完成、缺口、风险、解释、趋势、关联、建议和成就卡片。
- 每日最多输出三个最有价值洞察和三个小行动。
- 为安全升级、异常波动和周期相关结论增加证据门槛。

验收：固定输入得到固定结果；缺记录主要降低置信度，不自动判红。

## M4：日报与本地驾驶舱（已完成）

开发内容：

- 生成统一 dashboard JSON。
- 实现响应式本地驾驶舱。
- 实现 1080x1920 日报 PNG、HTML 和归档 PDF。
- 建立日期目录、报告索引和幂等生成机制。

验收：桌面和手机尺寸无重叠；PNG/PDF 关键数据一致；重复生成不会创建冲突记录。

## M5：周报与月报（已完成）

开发内容：

- 周报聚焦七日行为结构和可持续性。
- 月报聚焦身体变化、目标进展、长期行动和体检事项。
- 引入 21-28 天基线和 8-12 周趋势门槛。
- 数据不足时自动降级为记录摘要。

验收：三类报告拥有不同数据模型和布局，不是同一模板替换日期范围。

## M6：OpenClaw 交付闭环（已完成）

开发内容：

- 固化 CLI/JSON 调用契约。
- 配置 OpenClaw cron：补录、日报、周报、月报。
- 返回文本摘要和本地附件路径，由 OpenClaw 负责发送。
- 验证飞书和 QQ 的附件能力与失败重试。

验收：不配置独立模型 API，也能由 OpenClaw 完成记录、定时触发和报告发送。

## M7：发布准备（已完成）

开发内容：

- 数据迁移、备份恢复、隐私审计和安全测试。
- 构建匿名样例数据和端到端测试。
- 校验 Skill 本地安装、升级和卸载。
- 打包时扫描并拒绝数据库、密钥、记录和报告文件。

验收：新用户可从空目录初始化；升级不丢数据；发布包不包含私人信息。

## v0.1.0 发布状态

M0-M7 已形成一条完整本地链路：渠道消息由 OpenClaw 进入，BodyNote 负责记录、分析、报告、备份和隐私检查，OpenClaw 再按结构化附件清单完成发送。真实飞书/QQ 账号的最终路由仍需在安装环境中通过 `openclaw cron show` 和实际收件确认。
