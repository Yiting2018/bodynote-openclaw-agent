# BodyNote 产品契约

这些规则是 OpenClaw 对话层、BodyNote 领域处理器、确定性分析和报告渲染之间的稳定边界。

## 1. 意图与领域 Handler 契约

一个 OpenClaw 对话 Agent 调用一个确定性的 BodyNote Intent Router。Router 选择领域 Handler；Handler 是处理器，不是自治语言模型 Agent。

目标路由信封包含：

| 字段 | 含义 |
| --- | --- |
| `intent` | 记录、纠正、查询、分析、报告或食物库管理。 |
| `event_type` | 记录或纠正数据时使用的标准事件类型。 |
| `handler` | 被选中的领域 Handler。 |
| `required_fields` | 保存前必须具备的字段。 |
| `confidence` | 0–1 的抽取置信度。 |
| `ambiguities` | 竞争 Handler、未解析引用或含糊字段。 |
| `occurred_at` | 带时区的实际发生时间。 |
| `occurred_at_source` | 明确、相对、估算、消息时间或睡眠日期。 |

模糊的修改或删除必须先解析到唯一事件。包含多个可记录领域的消息必须暴露候选 Handler，不能静默假装没有歧义。

当前实现位于 `src/bodynote_agent/handlers/`。`MedicalReportHandler` 生成医疗报告事件；`CorrectionHandler` 生成更新或软删除命令，不创建伪造的 correction 健康事件。

## 2. 时间语义契约

按以下顺序解析发生时间：

1. 明确日期和时钟时间。
2. “1 小时前”“半小时前”等相对时长。
3. “昨天”“今早”等相对日期或模糊时段。
4. 消息接收时间。

餐次类型和发生时间相互独立。“早餐”可以确定餐次，但不能把当天消息强行改写为 08:00。显式 `--at` 覆盖自然语言推断。

完成的整夜睡眠归属于醒来的本地日期。没有醒来时间时显示“昨夜”，不虚构具体时钟。未来睡眠目标不能保存为已完成睡眠；午睡保留实际发生时间。

“今天很早”等模糊时段暂按 06:30 锚定，同时返回 `fuzzy_occurred_at` 歧义和 warning，供上层选择是否追问；不能伪装成用户提供的精确时间。

## 3. 评分与置信度契约

`health_score` 表示相对个人目标、个人基线和安全规则的状态；`confidence` 表示证据覆盖和可靠性。

- 缺数据降低置信度，不自动降低状态。
- 记录完整度不能贡献健康状态分。
- 缺少可用目标或个人基线时，该模块状态为未知。
- 明显偏离目标可以限制绿色展示，但不自动等同医疗风险。
- 紧急安全信号覆盖显示分数。
- 相同标准化输入必须产生相同评分、标题、洞察和行动。

## 4. 报告展示契约

BodyNote 有两个主要产品界面：

- Dashboard：完整交互式历史、趋势、食物库和纠错上下文；选定范围内不静默截断事件。
- 移动端 PNG：宽度固定为 1080px，高度随内容增长；在一张连续长图中展示全部当日事件摘要、全部入选洞察和全部入选行动，不分页、不生成续图。

HTML 和 PDF 只保留为可选导出格式，不作为独立产品界面，也不能通过事件数量切片制造数据丢失的错觉。医疗原文件和原始 JSON 只留在本地详情中，报告仅展示自然语言摘要。
